11.2.0 DCS Communication Systems

Like PLC systems, distributed control systems (DCS) have multiple communication paths. Figure 30 shows a typical DCS implementation. Note that there is a local data highway connecting the various elements on the system, with connection to higher-level systems. Each of these data paths has different requirements.

Instrumentation has generally been wired to the I/O subsystems using a star topology with a true master-slave relationship. With the advent of Smart instruments, connections are now being made using bus-type I/O networks. Because the DCS is generally interrupt-driven, the response times for these I/O channels are somewhat greater than in the case of the PLC. One vendor, Fisher Controls International, has joined with Allen Bradley (AB) to design a system that uses the AB remote I/O communication channel to connect AB I/O racks to Fisher Provox® control systems.

The local data highway or control LAN is generally proprietary, although standardization is being attempted. While MAP appears to be the leading solution, it falls short in redundancy and response time. The Mini-MAP version overcomes some of the speed objections and has been used in a redundant version. However, most systems retain proprietary networks to protect the installation base, since most vendors have a large installed base of equipment that uses these proprietary protocols. Typically, these control LANs use a deterministic access method and operate at moderately high speeds. Like PLC peer-to-peer networks, performance requirements can be lowered by embedding or hard coding as much information as possible in the devices during configuration and restricting message variability.

MAP and Mini-MAP have achieved the greatest penetration where higher-level networks are bridged to general-purpose systems, and to link separate DCSs to each other. Before the availability of these network links, the most common method of PLC-to-DCS communication was direct I/O-to-I/O wiring, as shown in Figure 31. Response speeds were high, but so were the wiring costs. The reliability, with so many connections, was generally low.

Like PLCs, DCSs also have gateways to higher-level systems. Most DCSs support gateways to token ring or token bus systems, with many offering TCP/IP gateways. Coupled with the OSI-compliant gateways to MAP-like stacks, full interconnectivity generally can be achieved even though a number of gateways may be required.
